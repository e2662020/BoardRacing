# MOB导播系统 - 直播包装编辑器需求文档

> 本文档专门用于指导另一位AI开发直播包装编辑器模块
> 版本：v1.0
> 日期：2026-03-22

---

## 一、模块定位

直播包装编辑器是MOB导播系统的**核心功能模块**，用于让设计师创建、编辑和管理直播过程中使用的视觉包装（如比分板、运动员信息卡、赛事标题等）。

### 1.1 用户角色
- **主要用户**：设计师
- **次要用户**：导播（调用包装）、管理员（管理所有包装）

### 1.2 入口位置
- 左侧导航栏 → "直播包装"
- 点击后进入包装管理列表页
- 从列表页可进入包装编辑器

---

## 二、功能架构

```
直播包装模块
├── 管理列表页
│   ├── 新建包装
│   ├── 编辑包装（进入编辑器）
│   ├── 删除包装
│   ├── 打标签
│   └── 搜索/筛选
│
└── 包装编辑器（核心）
    ├── 画布区域（16:9）
    ├── 左侧组件库
    ├── 右侧属性面板
    ├── 底部数据表面板
    ├── 底部节点编辑器（标签页切换）
    └── 顶部工具栏
```

## 三、管理列表页

### 3.1 页面布局
- 标准后台管理界面
- 左侧导航栏常驻
- 右侧为包装卡片列表

### 3.2 功能需求

#### 3.2.1 包装卡片展示
- 卡片形式展示每个包装
- 显示信息：
  - 包装名称
  - 预览缩略图（最后保存状态的截图）
  - 标签（多个标签以tag形式展示）
  - 创建时间
  - 最后修改时间
  - 创建者

#### 3.2.2 新建包装
- 点击"新建包装"按钮
- 弹出对话框，输入：
  - 包装名称（必填）
  - 标签（可选，可多选或新建）
  - 描述（可选）
- 确认后进入编辑器，创建空白画布

#### 3.2.3 编辑包装
- 点击卡片进入编辑器
- 加载该包装的所有数据

#### 3.2.4 删除包装
- 卡片上有删除按钮（需二次确认）
- 或批量选择后批量删除

#### 3.2.5 标签管理
- 支持为包装添加/移除标签
- 支持按标签筛选
- 标签颜色可自定义

## 四、包装编辑器（核心）

### 4.1 整体布局

```
┌─────────────────────────────────────────────────────────────┐
│  顶部工具栏（保存、预览、撤销、重做、导出等）                    │
├──────────┬──────────────────────────────┬───────────────────┤
│          │                              │                   │
│  左侧    │                              │    右侧           │
│  组件库  │        画布区域（16:9）        │    属性面板        │
│          │      1920x1080 可视化编辑      │                   │
│          │                              │                   │
├──────────┴──────────────────────────────┴───────────────────┤
│  底部标签页切换：【数据表面板】|【节点编辑器】                   │
│  （默认显示数据表面板，可切换至节点编辑器）                      │
└─────────────────────────────────────────────────────────────┘
```

### 4.2 画布区域（核心编辑区）

#### 4.2.1 画布规格
- **比例**：16:9
- **分辨率**：1920x1080（设计时以像素为单位）
- **背景**：可设置背景色或背景图
- **网格**：可选显示网格辅助线（可开关）

#### 4.2.2 交互操作
- **拖拽**：组件可在画布内自由拖拽
- **缩放**：支持画布缩放（50%-200%）
- **多选**：按住Shift或框选多选组件
- **对齐**：支持对齐辅助线（类似Figma）
- **撤销/重做**：支持操作历史

### 4.3 左侧组件库

#### 4.3.1 基础组件
必须提供以下基础组件：

| 组件类型 | 功能描述 | 可配置属性 |
|---------|---------|-----------|
| 文本框 | 显示文字内容 | 字体、大小、颜色、对齐、行高 |
| 图片 | 显示图片 | 图片源、填充模式、圆角 |
| 矩形 | 装饰性矩形 | 背景色、边框、圆角、阴影 |
| 圆形 | 装饰性圆形 | 背景色、边框、阴影 |
| 线条 | 分割线或装饰线 | 颜色、粗细、样式（实线/虚线） |
| 容器(Div) | 可包含其他组件 | 背景、边框、内边距、布局方式 |

#### 4.3.2 预设组件
提供一些常用的预设包装组件：
- 运动员信息卡模板
- 比分板模板
- 赛事标题模板
- 排名列表模板

#### 4.3.3 自定义组件
- 设计师可以将设计好的多个组件组合保存为自定义组件
- 自定义组件可在全局使用
- 支持文件夹分类管理

#### 4.3.4 拖拽交互
- 从组件库拖拽组件到画布
- 拖入容器后自动成为容器的子元素
- 组件拖入容器后自动吸附定位（默认左上角）

### 4.4 右侧属性面板

#### 4.4.1 基础属性
选中组件后显示：

**位置与尺寸**
- X坐标
- Y坐标
- 宽度
- 高度
- 旋转角度

**外观样式**
- 背景色/背景图
- 边框（颜色、粗细、圆角）
- 阴影
- 透明度
- 混合模式

**文本样式**（文本组件特有）
- 字体（系统字体列表）
- 字号
- 字重
- 颜色
- 对齐方式（左/中/右/两端）
- 行高
- 字间距

#### 4.4.2 动画设置
- **入场动画**：淡入、滑入、缩放等
- **出场动画**：淡出、滑出等
- **持续动画**：呼吸、闪烁等
- **动画参数**：持续时间、延迟、缓动函数

#### 4.4.3 数据绑定
- 显示当前组件已绑定的数据变量
- 支持解绑
- 显示绑定数据的预览值

#### 4.4.4 排序规则（列表组件特有）
- 可设置按数据表的某一列排序
- 支持升序/降序
- 支持FLIP动画选项（使文本跟随容器排序时产生平滑过渡）

### 4.5 底部数据表面板

#### 4.5.1 面板功能
- 以表格形式展示系统数据表内容
- 支持搜索和筛选
- 显示列名和数据

#### 4.5.2 数据绑定交互（核心功能）

**单单元格绑定**
1. 用户在数据表中选中一个单元格
2. 拖拽该单元格到画布上的组件
3. 系统自动：
   - 创建一个新的数据绑定变量
   - 自动命名（如：athlete_name_001）
   - 将该变量绑定到组件的对应属性（如文本内容）
4. 组件实时显示该单元格的值

**多单元格列表绑定**
1. 用户在数据表中选中多个单元格（如一行或多行）
2. 拖拽到画布
3. 系统自动：
   - 识别为列表数据
   - 创建列表绑定
   - 按选中数量自动创建对应数量的文本框/组件
   - 每个组件自动绑定到列表的对应项

#### 4.5.3 数据预览
- 绑定后组件显示实际数据值
- 数据表更新时，画布上的组件实时更新

### 4.6 底部节点编辑器（标签页切换）

#### 4.6.1 入口
- 底部面板有两个标签页：【数据表面板】|【节点编辑器】
- 点击切换

#### 4.6.2 界面风格
- 类似Blender的节点编辑器
- 画布式布局，可拖拽、缩放
- 节点之间用连线连接

#### 4.6.3 节点类型

**基础逻辑节点**
| 节点名称 | 功能 | 输入 | 输出 |
|---------|------|------|------|
| If | 条件判断 | condition, true_value, false_value | result |
| Compare | 比较运算 | a, b, operator(==, !=, >, <, >=, <=) | boolean |
| And/Or/Not | 逻辑运算 | inputs | boolean |
| ForEach | 遍历列表 | list | item, index |
| Math | 数学运算 | a, b, operator | result |
| Variable | 变量定义/读取 | value（可选） | value |

**功能节点**
| 节点名称 | 功能 | 参数 |
|---------|------|------|
| SetData | 修改数据表 | table, row, column, value |
| ShowComponent | 显示组件 | component_id |
| HideComponent | 隐藏组件 | component_id |
| ChangeImage | 更改图片 | component_id, image_url |
| ChangeText | 更改文本 | component_id, text |
| PlayAnimation | 播放动画 | component_id, animation_name |
| TriggerEvent | 触发事件 | event_name, data |
| Delay | 延迟执行 | duration |

**数据节点**
| 节点名称 | 功能 |
|---------|------|
| GetData | 获取数据表某单元格值 |
| GetBoundData | 获取组件绑定的数据 |
| Constant | 常量值（字符串、数字、布尔值） |

#### 4.6.4 节点交互
- 从节点边缘拖拽创建连线
- 连线连接不同节点的输入/输出端口
- 支持删除节点和连线
- 支持复制粘贴节点
- 节点可折叠/展开

#### 4.6.5 执行逻辑
- 节点图按照数据流方向执行
- 支持事件触发（如：数据变化时自动执行某分支）
- 支持手动触发测试

## 五、数据存储与自动保存

### 5.1 自动保存
- 编辑器内所有操作自动保存
- 保存间隔：每5秒或每次操作后
- 保存内容：
  - 画布上所有组件及其属性
  - 组件之间的层级关系
  - 数据绑定关系
  - 节点编辑器逻辑
  - 动画设置

### 5.2 工程文件结构
```json
{
  "id": "uuid",
  "name": "包装名称",
  "description": "描述",
  "tags": ["标签1", "标签2"],
  "canvas": {
    "width": 1920,
    "height": 1080,
    "background": "#000000"
  },
  "components": [
    {
      "id": "comp_001",
      "type": "text",
      "name": "运动员姓名",
      "x": 100,
      "y": 100,
      "width": 200,
      "height": 50,
      "style": { /* 样式属性 */ },
      "animation": { /* 动画配置 */ },
```

- 导出为图片（PNG/JPG）

## 七、技术实现建议

### 7.1 推荐开源库

| 功能 | 推荐库 | 说明 |
|------|--------|------|
| 画布渲染 | Fabric.js / Konva.js | 2D画布操作 |
| 拖拽系统 | @dnd-kit | React拖拽方案 |
| 节点编辑器 | Rete.js / ReactFlow | 节点图编辑器 |
| 动画 | GSAP / Framer Motion | 高性能动画 |
| 数据表 | LuckySheet | 已集成在项目中 |

### 7.2 性能要求
- 画布上支持至少50个组件流畅操作
- 节点编辑器支持至少30个节点流畅运行
- 数据绑定更新延迟 < 100ms

### 7.3 代码规范
- 使用TypeScript
- 组件化开发
- 状态管理使用Zustand（项目已集成）
- 遵循项目已有的代码风格
